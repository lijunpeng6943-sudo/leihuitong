# 方言学习平台 - 最终版本

## 项目介绍
这是一个功能完善的方言学习平台，支持雷州方言和惠来方言的学习和管理。本版本实现了真正的服务器端数据存储，解决了之前版本需要手动下载文件的问题。

## 核心功能
- ✅ 按分类管理方言词汇
- ✅ 音频播放功能
- ✅ 词汇的添加、编辑、删除
- ✅ 数据的导入导出（JSON/Excel）
- ✅ 真正的服务器端数据存储
- ✅ 响应式设计，支持移动端

## 技术架构
- **前端**: HTML5, CSS3, JavaScript, Tailwind CSS
- **后端**: Node.js HTTP服务器
- **数据存储**: JSON文件
- **依赖库**: Font Awesome, SheetJS (Excel处理)

## 解决的问题
在之前的版本中，每次点击"保存数据"都会触发文件下载，用户需要手动保存文件并替换原始的data.json文件。

**最终解决方案**:
- 添加了Node.js后端服务器
- 实现了RESTful API接口
- 前端通过API直接读写服务器上的data.json文件
- 无需用户手动下载和替换文件

## 如何使用

### 1. 启动服务器
```bash
# 在项目目录下运行
node server.js
```

服务器会在 http://localhost:3000 启动

### 2. 访问应用
打开浏览器，访问 http://localhost:3000

### 3. 开始使用
- 查看现有的方言词汇
- 点击音频按钮播放发音
- 使用"管理词汇"功能添加新词汇
- 点击"保存数据"按钮将数据保存到服务器

## API接口
- **GET /api/data** - 获取所有方言数据
- **POST /api/data** - 保存方言数据到服务器

## 数据备份机制
- 自动备份到浏览器本地存储
- 服务器保存失败时自动降级为文件下载
- 多重保障确保数据不丢失

## 注意事项
1. 请确保Node.js已安装在您的系统上
2. 服务器启动后请勿关闭终端窗口
3. 如果端口3000被占用，可以修改server.js中的PORT配置
4. 数据文件(data.json)会自动创建和更新

## 故障排除
- **服务器启动失败**: 检查端口是否被占用
- **数据加载失败**: 检查服务器是否正常运行
- **保存失败**: 检查网络连接和服务器状态

如果遇到问题，可以查看浏览器控制台和服务器日志获取详细信息。
